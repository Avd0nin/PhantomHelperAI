ТЗ ПРОЕКТА PHANTOMHELPER AI

1. ОБЩЕЕ ОПИСАНИЕ  
Проект представляет собой веб-приложение для помощи в обучении с использованием искусственного интеллекта.  
Система объединяет генерацию конспектов, чат, тестирование и контесты в одном интерфейсе.  
Все действия пользователя сохраняются и используются для формирования истории обучения.

2. ЦЕЛЬ ПРОЕКТА  
Создать платформу, позволяющую пользователю:  
- изучать учебный материал  
- получать объяснения через AI  
- проходить тесты  
- решать задачи  
- отслеживать свой прогресс  

3. ФУНКЦИОНАЛ

3.1 АВТОРИЗАЦИЯ  
- регистрация пользователя  
- вход в аккаунт  
- хранение сессии  
- выход из системы  

3.2 ГЕНЕРАЦИЯ КОНСПЕКТОВ  
- ввод темы, предмета, уровня  
- генерация структурированного текста  
- наличие определений, примеров  
- сохранение конспекта в базе данных  
- просмотр ранее созданных конспектов  

3.3 AI ЧАТ  
- ввод вопроса пользователем  
- генерация ответа системой  
- хранение истории сообщений  
- возможность продолжения диалога  

3.4 ТЕСТЫ  
- генерация тестов по теме  
- поддержка вариантов ответа  
- поддержка открытых вопросов  
- проверка ответов  
- отображение результата  
- сохранение попыток  

3.5 КОНТЕСТЫ  
- генерация задач  
- настройка сложности  
- настройка количества задач  
- наличие условий и примеров  
- наличие таймера  
- запуск контеста  

3.6 ПРОВЕРКА КОДА  
- ввод решения пользователем  
- выбор языка (Python / C++)  
- запуск кода  
- сравнение вывода с ожидаемым  
- возврат результата (OK / ошибка)  
- отображение результатов тестов  

3.7 ПРОФИЛЬ ПОЛЬЗОВАТЕЛЯ  
- отображение статистики  
- количество конспектов  
- количество тестов  
- количество попыток  
- история действий  

4. АРХИТЕКТУРА  

Frontend: HTML, CSS, JavaScript, шаблоны  
Backend: Python, Flask  
Логика: модуль AICore  
База данных: SQLite  
Модуль проверки кода: subprocess  

5. СТРУКТУРА БАЗЫ ДАННЫХ  

users  
summaries  
chat_threads  
chat_messages  
tests  
test_attempts  
contests  
contest_attempts  

6. ПОТОК ДАННЫХ  

Пользователь → backend → AI → база данных → ответ  
Код → запуск → тесты → результат  

7. НЕФУНКЦИОНАЛЬНЫЕ ТРЕБОВАНИЯ  

- корректная работа интерфейса  
- сохранение данных  
- обработка ошибок  
- стабильный запуск кода  

8. ДАЛЬНЕЙШЕЕ РАЗВИТИЕ  

- мобильная адаптация  
- улучшение AI  
- расширение задач  
- улучшение безопасности  
- аналитика пользователя
