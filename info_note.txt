ПОЯСНИТЕЛЬНАЯ ЗАПИСКА
к проекту PhantomHelper AI

1. ВВЕДЕНИЕ

В современном обучении учащиеся сталкиваются с рядом проблем: отсутствует единая среда для изучения материала, объяснения сложных тем часто недостаточно понятны, а инструменты для проверки знаний разрознены.

Для подготовки к одной теме пользователю приходится использовать несколько сервисов одновременно: учебные материалы, онлайн-решебники, тестовые платформы и сайты с задачами. Это усложняет процесс обучения и снижает его эффективность.

Проект PhantomHelper AI направлен на решение этой проблемы путем объединения всех ключевых элементов обучения в одной системе.

2. НАЗНАЧЕНИЕ ПРОЕКТА

PhantomHelper AI представляет собой веб-приложение, предназначенное для помощи в обучении с использованием технологий искусственного интеллекта.

Основная задача системы — обеспечить полный цикл обучения:
- изучение темы
- получение объяснений
- проверка знаний
- практическое применение
- сохранение результатов

3. ФУНКЦИОНАЛЬНЫЕ ВОЗМОЖНОСТИ

3.1 Генерация конспектов
Система позволяет пользователю вводить предмет, тему и уровень сложности. На основе этих данных генерируется структурированный учебный материал, включающий:
- определения
- правила
- примеры
- пояснения
- типичные ошибки

Конспекты сохраняются в базе данных и доступны для повторного просмотра.

3.2 AI-чат
Чат позволяет пользователю задавать вопросы по теме и получать развернутые ответы. Система поддерживает:
- диалоговый режим
- сохранение истории сообщений
- использование контекста (например, конспекта)

3.3 Тестирование
Система генерирует тесты по заданной теме. Поддерживаются:
- вопросы с вариантами ответа
- открытые вопросы
- автоматическая проверка
- сохранение результатов попыток

3.4 Контесты
Контесты представляют собой набор задач по программированию. Пользователь может:
- выбрать сложность
- выбрать количество задач
- проходить контест с таймером
- получать результат

3.5 Проверка кода
Система поддерживает запуск пользовательского кода:
- Python выполняется через интерпретатор
- C++ компилируется и запускается
- результат сравнивается с ожидаемым выводом
- возвращается вердикт (OK, ошибка и др.)

3.6 Профиль пользователя
Профиль хранит:
- конспекты
- тесты
- чаты
- контесты
- результаты

Это позволяет отслеживать прогресс пользователя.

4. АРХИТЕКТУРА СИСТЕМЫ

Система состоит из следующих компонентов:

Frontend:
- HTML, CSS, JavaScript
- шаблоны Jinja2

Backend:
- Python, Flask
- обработка маршрутов и API

AI-модуль:
- AICore
- генерация контента и ответов

База данных:
- SQLite
- хранение всех данных

Модуль проверки кода:
- subprocess
- временные файлы
- выполнение программ

5. СТРУКТУРА БАЗЫ ДАННЫХ

Основные таблицы:
- users (пользователи)
- summaries (конспекты)
- chat_threads (чаты)
- chat_messages (сообщения)
- tests (тесты)
- test_attempts (попытки тестов)
- contests (контесты)
- contest_attempts (попытки контестов)

Все таблицы связаны через user_id, что позволяет хранить данные отдельно для каждого пользователя.

6. ПОТОК ДАННЫХ

Общий процесс:
Пользователь отправляет запрос → backend обрабатывает его → при необходимости обращается к AI → результат сохраняется в базе данных → ответ возвращается пользователю.

Для проверки кода:
код пользователя → выполнение → сравнение с тестами → формирование результата.

7. НЕФУНКЦИОНАЛЬНЫЕ ТРЕБОВАНИЯ

- стабильная работа системы
- корректная обработка ошибок
- сохранение пользовательских данных
- защита от некорректного ввода
- ограничение времени выполнения программ

8. ПЕРСПЕКТИВЫ РАЗВИТИЯ

- адаптация под мобильные устройства
- улучшение качества генерации AI
- расширение базы задач
- внедрение безопасного sandbox для кода
- развитие системы аналитики прогресса пользователя

ЗАКЛЮЧЕНИЕ

PhantomHelper AI представляет собой комплексное решение для обучения, объединяющее теорию, объяснение, проверку и практику. Проект демонстрирует интеграцию современных технологий искусственного интеллекта с веб-разработкой и может быть расширен до полноценного образовательного сервиса.
